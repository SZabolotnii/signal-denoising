# Plan: Гібридна модель знешумлення DSGE + U-Net

## Мета проєкту

Інтегрувати математичний апарат розкладу в просторі з порідним елементом (DSGE — Decomposition in Space with Generating Element, простір Кунченка) з існуючою U-Net архітектурою для покращення якості знешумлення цифрових сигналів QPSK/FSK в умовах гаусівських та негаусівських завад.

---

## 1. Теоретичне обґрунтування

### 1.1 Суть підходу DSGE (простір Кунченка)

Порідний вектор X (чистий сигнал) наближається узгодженим вектором Y, побудованим через нелінійні базисні функції від зашумлених спостережень:

```
X = k₀ + Σᵢ kᵢ · φᵢ(x̃) + Z
```

де:
- X — порідний вектор (чистий сигнал, до якого ми наближаємось)
- x̃ — зашумлені спостереження (вхідні дані)
- φᵢ(·) — нелінійні базисні функції
- K = {k₁, ..., kₛ} — оптимальні коефіцієнти розкладу
- Z = X − Y — залишкова похибка (шум)
- S — порядок апроксимації

Оптимальні коефіцієнти знаходяться із системи:

```
KF = B
```

де:
- F — матриця центрованих корелянтів: Fᵢⱼ = E{[φᵢ(x̃) − Ψᵢ][φⱼ(x̃) − Ψⱼ]}
- B — вектор взаємних кореляцій: Bᵢ = E{[x − Ψ₀][φᵢ(x̃) − Ψᵢ]}
- Ψᵢ = E{φᵢ(x̃)}, Ψ₀ = E{x}

Ключова властивість: при S → ∞ похибка апроксимації → 0 (повний розклад у стохастичний функціональний ряд).

### 1.2 Парадигма застосування до знешумлення

| Поняття Ku-простору | Роль у знешумленні |
|---|---|
| Порідний вектор X | Чистий (очікуваний) сигнал |
| Узгоджений вектор Y | Наближення чистого сигналу |
| Зашумлені спостереження x̃ | Вхідний зашумлений сигнал |
| Базисні функції φᵢ(x̃) | Нелінійні перетворення зашумленого сигналу |
| Похибка Z = X − Y | Залишковий шум після знешумлення |
| Мінімізація ‖Z‖² | Мінімізація MSE знешумлення |

### 1.3 Обґрунтування гібридного підходу

Згідно з результатами HAR-статті (Zabolotnii et al.), DSGE-ознаки демонструють **комплементарність** з традиційними ознаками:
- Самі DSGE-ознаки (6 фіч): 84.36% accuracy
- Традиційні ознаки (561 фіча): 95.72% accuracy
- Гібрид (561 + 6 DSGE): **95.86%** — покращення навіть при ceiling effect

Для знешумлення: STFT-спектрограма — "традиційні ознаки", DSGE — додаткова інформація про статистичну структуру сигналу, яку STFT не захоплює. Особливо для негаусівських шумів (поточний MSE = 0.126, є потенціал покращення).

## 2. Архітектурні рішення MVP

### 2.1 Базисні функції

Відповідно до результатів HAR-статті, дробово-степеневий базис показав найкращу індивідуальну точність (82.59%). Використовуємо:

```
φᵢ(x̃) = sign(x̃) · |x̃|^pᵢ
```

Значення степенів для MVP (S=3):
- p₁ = 0.5 (√|x| зі знаком — пригнічує великі викиди, підсилює малі)
- p₂ = 1.5 (проміжна нелінійність)
- p₃ = 2.0 (квадратична — захоплює енергетичну інформацію)

Обґрунтування S=3: grid search у HAR-статті показав, що S=3 — оптимальний порядок, який дає результат на рівні ансамблю 12 функцій.

Збереження знаку (sign(x)) критично важливе для радіосигналів QPSK/FSK, де фаза несе інформацію.

### 2.2 Породжуючий елемент

Два породжуючих елементи (M=2):
- X⁽⁰⁾ — усереднений нормалізований чистий QPSK-сигнал
- X⁽¹⁾ — усереднений нормалізований чистий FSK-сигнал

**Процедура формування** (аналогічна HAR-статті, секція 4.2):
1. Нормалізація всіх чистих сигналів: E{x} = 0, Var{x} = 1
2. Розділення на QPSK та FSK підмножини
3. Обчислення поелементного середнього для кожної підмножини

**Відоме обмеження:** усереднення сигналів з різними несучими частотами (10–100 Гц) може давати малоінформативний породжуючий елемент (≈ 0). Це буде перевірено в Експерименті 1. Якщо підтвердиться — перехід на кластеризацію за частотою (Розділ 5.2).

### 2.3 Точка інтеграції з U-Net

Обрано **Точку А: додаткові канали на вході U-Net** (прямий аналог Experiment 4 з HAR).

Поточний пайплайн:
```
x̃(t) → [STFT] → mag(f,t) [1 канал] → [U-Net] → mask → [ISTFT] → ŷ(t)
```

Новий пайплайн:
```
x̃(t) → [STFT] → mag(f,t)           ─┐
x̃(t) → [φ₁(x̃)] → [STFT] → mag₁(f,t) ─┤
x̃(t) → [φ₂(x̃)] → [STFT] → mag₂(f,t) ─┼→ [concat: 4 канали] → [U-Net*] → mask → [ISTFT] → ŷ(t)
x̃(t) → [φ₃(x̃)] → [STFT] → mag₃(f,t) ─┘
```

Зміна в U-Net: перший згортковий шар приймає 4 канали замість 1:
```python
# Було:
nn.Conv2d(1, 16, kernel_size=3, padding=1)
# Стало:
nn.Conv2d(1 + S, 16, kernel_size=3, padding=1)  # S=3 → 4 канали
```

### 2.4 DSGE-ознаки у T-F домені

Для кожної базисної функції φᵢ:
1. Обчислити φᵢ(x̃ₙ) поелементно від часового сигналу
2. Виконати STFT від φᵢ(x̃) з тими самими параметрами (nperseg=128, noverlap=96)
3. Взяти амплітудну спектрограму |STFT{φᵢ(x̃)}|
4. Конкатенувати з основною спектрограмою як додатковий канал

Це зберігає теоретичну коректність: φᵢ обчислюються від спостережень (зашумленого сигналу), потім переводяться у T-F домен для сумісності з U-Net.


### 3.2 Етап 1: DSGE-модуль (`models/dsge_layer.py`)

Реалізувати клас `DSGEFeatureExtractor`:

```python
class DSGEFeatureExtractor:
    """
    Обчислення DSGE-ознак для знешумлення сигналів.
    
    Реалізує:
    1. Формування породжуючого елемента з тренувальних даних
    2. Обчислення базисних функцій φᵢ(x̃)
    3. Розв'язання системи KF = B з регуляризацією Тихонова
    4. Генерацію DSGE-спектрограм для U-Net
    """
```

Ключові методи:
- `fit(clean_signals, noisy_signals)` — обчислення K, Ψᵢ, Ψ₀ на тренувальних даних
- `compute_basis(noisy_signal)` → масив [S, T] значень φᵢ(x̃ₙ)
- `compute_dsge_spectrograms(noisy_signal)` → тензор [S, F, T'] спектрограм від φᵢ(x̃)
- `reconstruct(noisy_signal)` → узгоджений вектор Y (пряме DSGE-наближення)

Параметри:
- `basis_type`: 'fractional' (sign(x)|x|^p)
- `powers`: [0.5, 1.5, 2.0]
- `tikhonov_lambda`: 0.01
- `stft_params`: {'nperseg': 128, 'noverlap': 96, 'fs': 1024}

### 3.3 Етап 2: Гібридна модель (`models/hybrid_unet.py`)

```python
class HybridDSGE_UNet(nn.Module):
    """
    U-Net автоенкодер з DSGE-каналами на вході.
    
    Вхід: (B, 1+S, F, T') — конкатенація STFT-спектрограми та S DSGE-спектрограм
    Вихід: (B, 1, F, T') — маска знешумлення
    """
```

Архітектура ідентична поточному UnetAutoencoder, єдина зміна — `in_channels = 1 + S`.

### 3.4 Етап 3: Тренування (`train/training_hybrid.py`)

Процес тренування:
1. Завантаження clean/noisy датасетів
2. Ініціалізація DSGEFeatureExtractor, виклик `fit()` на тренувальній підмножині
3. Препроцесинг: для кожного сигналу обчислити [mag, dsge_mag₁, dsge_mag₂, dsge_mag₃]
4. Тренування U-Net (4 канали на вході) з MSELoss, Adam, lr=1e-4
5. Логування в wandb

Розбиття даних: 70/15/15 (як у поточному training_uae.py).

### 3.5 Етап 4: Інференс та порівняння

Для кожного тестового сигналу:
1. Обчислити 4-канальний вхід (STFT + 3 DSGE-спектрограми)
2. Пропустити через навчену модель → маска
3. ISTFT → знешумлений сигнал
4. Порівняти MSE та SNR з baseline U-Net (1 канал)

### Експеримент 1: MVP — базова валідація гіпотези

**Мета:** чи додають DSGE-канали статистично значуще покращення MSE?

| Параметр | Значення |
|---|---|
| Базис | sign(x)\|x\|^p, p ∈ {0.5, 1.5, 2.0} |
| Порядок S | 3 |
| Породжуючий елемент | 2 (середній QPSK, середній FSK) |
| Регуляризація | Тихонов, λ = 0.01 |
| Архітектура U-Net | Поточна + зміна in_channels: 1 → 4 |
| Тренування | 30 епох, batch=32, Adam lr=1e-4 |
| Датасети | gaussian, non_gaussian |

Порівняння:

| Модель | Вхід | Очікуваний результат |
|---|---|---|
| Baseline U-Net | mag (1 канал) | MSE ≈ 0.042 (gauss), 0.126 (non-gauss) |
| DSGE-only | Y = k₀ + KΦ(x̃) | Оцінка ефективності чистого DSGE |
| Hybrid U-Net | mag + 3 DSGE (4 канали) | Очікуване покращення на non-gauss |

**Критерій успіху MVP:** статистично значуще зниження MSE на негаусівських шумах (p < 0.05 за t-test на тестовій вибірці).

**Мета:** знайти оптимальний тип базису для радіосигналів.

| Тип | Функції | Обґрунтування |

Для кожного типу: навчити окрему гібридну модель, порівняти MSE/SNR.

Компоненти для ablation:

## 5. Ризики та плани мітігації

### 5.1 Ризик: вироджений породжуючий елемент

**Опис:** усереднення QPSK/FSK сигналів з різними несучими (10–100 Гц) може давати ≈ 0.

**Перевірка:** в Експерименті 1 обчислити ‖X⁽ᵐ⁾‖ для кожного породжуючого елемента. Якщо ‖X⁽ᵐ⁾‖ < ε (порогове значення), елемент неінформативний.

**Мітігація:**
- Кластеризація чистих сигналів за несучою частотою (K-means, K=5–10 кластерів)
- Або: відмова від породжуючого елемента для ознак. Використовувати тільки φᵢ(x̃) → STFT як додаткові канали, без формування Y і обчислення εc. Тоді DSGE-канали — це просто нелінійні перетворення вхідного сигналу, а U-Net сам вчиться їх використовувати.

### 5.2 Ризик: DSGE-канали не додають інформації

**Опис:** STFT вже достатньо захоплює структуру сигналу, і φᵢ(x̃) не дають нового.

**Перевірка:** обчислити кореляцію між STFT-спектрограмою та DSGE-спектрограмами. Якщо кореляція > 0.95 — канали дублюють інформацію.

**Мітігація:**
- Спробувати інші базиси (тригонометричні, робастні)
- Спробувати іншу точку інтеграції (bottleneck conditioning замість вхідних каналів)
- Подавати DSGE-ознаки не як спектрограми, а як скалярні кондиціонери (εc) через FiLM-шар

### 5.3 Ризик: збільшення параметрів нечесно порівнюється

**Опис:** hybrid U-Net має більше параметрів через 4 вхідних канали → покращення може бути через додаткову ємність, а не через DSGE.

**Мітігація (fair comparison):**
- Порівняти з "Wider U-Net": U-Net із 1 каналом, але з enc1 = Conv2d(1, 20) замість Conv2d(1, 16), щоб матчити кількість параметрів
- Порівняти з "Random channels": U-Net із 4 каналами, де 3 додаткових — випадковий шум (а не DSGE). Якщо DSGE-варіант значно кращий — покращення від інформації, а не від ємності

### 5.4 Ризик: обчислювальна складність

**Опис:** обчислення 3 додаткових STFT збільшує час препроцесингу.

**Оцінка:** STFT одного сигналу (N=2144, nperseg=128) ≈ 0.1 мс. Додаткові 3 STFT ≈ 0.3 мс. Для batch=32 ≈ 10 мс. Некритично.

**Оцінка DSGE-обчислень:** φᵢ(x̃) — поелементні операції O(N·S). Для N=2144, S=3: ≈ 6432 операцій. Некритично.

## 6. Метрики та критерії успіху

### 6.1 Основні метрики

| Метрика | Формула | Напрямок |
|---|---|---|
| MSE | (1/N) Σ(x − ŷ)² | Менше = краще |
| SNR | 10·log₁₀(Σx² / Σ(x−ŷ)²) | Більше = краще |
| MAE | (1/N) Σ\|x − ŷ\| | Менше = краще |
| RMSE | √MSE | Менше = краще |

### 6.2 Критерії прийняття рішень

| Результат | Рішення |
|---|---|
| MSE hybrid < MSE baseline (p < 0.05) | Публікація: гібридний підхід ефективний |
| MSE hybrid ≈ MSE baseline | Аналіз: чому DSGE не додає інформації → зміна базису/точки інтеграції |
| MSE hybrid > MSE baseline | DSGE шкодить → аналіз кореляцій, можливий перегляд парадигми |

### 6.3 Baseline значення для порівняння

Поточні результати (з Таблиці 1 статті по знешумленню):

| Transformer | 0.3296 | 0.3435 | 1.81 dB | 3.78 dB |
| Wavelet | 0.132990 | 1.039383 | 5.78 dB | −1.55 dB |

Цільові значення для гібриду: MSE < 0.04 (gauss), MSE < 0.10 (non-gauss).

### 8.1 Поточна стаття (знешумлення)

Розширення розділу "Моделі для знешумлення" новим підходом: DSGE + U-Net як четвертий метод порівняння (або як покращення U-Net).

### 8.2 HAR-стаття (Zabolotnii et al.)

Прямий зв'язок через методологію DSGE. Ця робота розширює застосування Ku-простору від класифікації сенсорних даних до знешумлення радіосигналів.

### 8.3 Потенційна нова стаття

"Hybrid DSGE-UNet Denoising for Non-Gaussian Wireless Channels" — окрема публікація, якщо результати значимі. Наративна лінія: від аналітичного DSGE (теорія) → через STFT+U-Net (практика) → до гібриду (синтез).

### 8.4 Зв'язок з запланованою статтею про автоенкодери

README проєкту вказує на ідею порівняння 3 автоенкодерів (U-Net, ResNet, VAE). DSGE-шар можна додати до кожного з них і порівняти вплив → матеріал для ще однієї публікації.

## 9. Відкриті питання для подальшого дослідження

1. **Навчуваний DSGE-шар.** Замість фіксованих степенів p — зробити їх параметрами nn.Parameter і навчати end-to-end. Це з'єднає класичну теорію Ku-простору з deep learning.

2. **Локальний vs глобальний DSGE.** Обчислювати K у ковзному вікні (як STFT) замість глобально. Може бути ефективнішим для нестаціонарних сигналів.

3. **DSGE від спектрограми.** Замість φᵢ(x̃(t)) → STFT обчислювати φᵢ(mag(f,t)) безпосередньо на спектрограмі. Це зменшить обчислення (один STFT замість чотирьох).

4. **Фазова інформація.** Поточний U-Net працює тільки з амплітудою, фаза зберігається від зашумленого сигналу. DSGE-ознаки від фазової спектрограми можуть дати додаткову інформацію.

5. **Порівняння з іншими feature engineering підходами.** Scattering transform, Mel-spectrograms, CQT — як альтернативні "додаткові канали" для U-Net, baseline для оцінки специфічної цінності DSGE.
